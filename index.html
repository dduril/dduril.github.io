Things to remember

In the course of being a web developer, I find that there are just too many things to remember. We should be focused on how to solve problems and deliver solutions. But, so often, I find myself trying to recall little tidbits of code like the format specifiers for dates.

And so, this site will be an attempt to distill the things that I'm always trying to remember, but don't want to memorize - doctypes, HTML5 tags and attributes, CSS3 media queries, jQuery syntax, SQL differences between MySQL-MS SQL Server-Oracle, how to use Github, yeah - all those command-line gems with their switches and parameters for Rails or Node.

I suppose I could always just google these things, but I'd like to have my own personal site for these little treasures. And, maybe if someone else stumbles upon this site, they will find something useful here, too.
