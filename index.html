Things To Remember

In the course of being a Web Developer or now more commonly - Full Stack or Front End Developer - there are a lot of things to remember. Ofcourse, being a developer in any space can be overwhelming, for example data engineering or games programming. The proliferation and acceptance of open source software and github have certainly help alot in this regard. It is very easy to quickly find a library or module to help out with your project.

As a Web Developer, I've found this to be especially true. Every day brings another wave of blogs, newsletters, podcasts, LinkedIn articles, tweets and tutorials on how to build truly unique and interesting features into mobile and responsive apps. All of which makes being a developer in the web space a fun and rewarding, if not also, hectic and challenging endeavor with plenty of opportunity to continously learn the craft.

But, this constant learning of new frameworks and libraries, utilities and tools, patterns and opinions can also bog you down and leave you feeling overwhelmed. And sometimes, in all of this racing to keep up with the latest pattern or framework, you really find yourself focused on remembering different date format specifiers or some little bit of syntax, instead of focusing on bigger things - like solving the problem and delivering our solution.

And so, I'd like to focus this site on remembering some of those things - like doctypes, html5 tags and attributes, css3 and media queries, jQuery syntax and properties, database queries and server settings, how to use Github, yeah - and all of those Java, Python, Ruby and Node command-line gems with their options and parameters.

Much of this can be easily found on Google, MDN, MSDN, Stack Overflow and other great sites like A List Apart and Smashing Magazine. But, I'd also like to contribute my collection of web programming techniques and ideas based on many years of working in this field.

Then, I can get back to working on the bigger problems - knowing that I'll always know where to find that date format specifier.
